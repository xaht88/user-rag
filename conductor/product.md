# Product — RAG Chat Application

## 🎯 Product Vision

**RAG Chat Application** — интеллектуальная система диалогового взаимодействия с документами, использующая Retrieval-Augmented Generation (RAG) архитектуру для предоставления точных ответов с указанием источников.

### Problem Statement

Организации и специалисты работают с огромными объёмами документированной информации (PDF, DOCX, TXT, MD), но традиционные методы поиска не позволяют:
- Находить информацию семантически, а не по ключевым словам
- Получать обобщённые ответы на основе нескольких документов
- Отслеживать источники информации для верификации

### Solution Approach

Система автоматически:
1. Загружает документы и разбивает их на семантические чанки
2. Создаёт векторные эмбеддинги для каждого чанка
3. При запросе пользователя находит наиболее релевантные чанки
4. Генерирует ответ через LLM с цитированием источников

## 👥 Target User Personas

### 1. Аналитик / Исследователь
**Портрет:** Работает с объёмными отчётами, научными статьями, юридическими документами  
**Боли:** Тратит часы на ручной поиск информации в документах  
**Цели:** Быстро находить ответы, получать обобщения, проверять источники  
**Критерии успеха:** Экономия времени на 80%, точность ответов >90%

### 2. Юрист
**Портрет:** Работает с контрактами, прецедентами, нормативными актами  
**Боли:** Необходимость цитирования источников в отчётах  
**Цели:** Быстрый поиск по документам, точное цитирование  
**Критерии успеха:** Возможность указать конкретную страницу источника

### 3. Разработчик
**Портрет:** Изучает техническую документацию, API справочники  
**Боли:** Разрозненная документация, сложность навигации  
**Цели:** Быстрый поиск по технической документации  
**Критерии успеха:** Поддержка форматов Markdown, код в ответах

### 4. Мобильный пользователь
**Портрет:** Работает с документов на ходу, с телефона  
**Боли:** Ограниченный экран, сложность чтения больших документов  
**Цели:** Быстрый доступ к информации с мобильного  
**Критерии успеха:** Адаптивный интерфейс, быстрая загрузка

## ✨ Core Features

### F-01: Загрузка и обработка документов
- Поддержка форматов: PDF, DOCX, TXT, MD
- Максимальный размер файла: 50 МБ
- Автоматическая разбивка на чанки (512 токенов, перекрытие 50)
- Создание эмбеддингов через BERT-based модель
- Сохранение метаданных (имя, дата, страницы, чанки)
- Индикатор прогресса обработки

### F-02: Выбор и настройка LLM
- Поддержка OpenAI (GPT-4o, GPT-4o-mini)
- Поддержка Ollama (локальные модели: Llama2, Mistral)
- Сохранение выбора в рамках сессии
- Поддержка пользовательского API-ключа
- Уведомление при недоступности модели

### F-03: Диалоговый интерфейс
- Векторный поиск топ-5 релевантных чанков
- Формирование Context Window для LLM
- Отображение ответа в течение 30 секунд
- Показ источников: имя файла + номер страницы
- История диалога в рамках сессии
- Индикатор загрузки при обработке

### F-04: Управление документами
- Список загруженных документов с метаданными
- Удаление документа с очисткой из Vector Store (≤5 сек)
- Выбор активных документов для поиска
- Лимит: не более 10 документов на сессию

### F-05: Управление сессиями
- Изоляция данных сессий
- Автоматическая очистка при закрытии браузера
- Сохранение конфигурации LLM в рамках сессии

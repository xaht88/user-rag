# UI Концепции: RAG Веб-приложение

## Концепция 1: Минималистичная (Split-Panel)

**Описание**: Двухпанельный layout. Левая панель (30%) — список документов и загрузка. Правая панель (70%) — чат-интерфейс. Верхняя строка — выбор LLM.

**Стиль**: Белый фон, серые границы, акцент — синий (#2563EB). Шрифт Inter.

**Преимущества**:
- Максимальная концентрация на диалоге
- Быстрый доступ к документам без переключения экранов
- Хорошо работает на широких экранах

**Недостатки**:
- На мобильных требует переключения между панелями
- Мало места для метаданных документов

---

## Концепция 2: Корпоративная (Dashboard)

**Описание**: Верхний navbar с выбором LLM и профилем. Левый sidebar (250px) — навигация и документы. Центральная область — чат. Правый sidebar (200px) — источники текущего ответа.

**Стиль**: Тёмно-синий (#1E3A5F) + белый, иконки Material Design.

**Преимущества**:
- Привычный корпоративный UX
- Источники всегда видны рядом с ответом
- Масштабируется для добавления новых функций

**Недостатки**:
- Перегружен для MVP
- На мобильных sidebars скрываются, теряется контекст

---

## Концепция 3: Мобильно-ориентированная (Tab-based)

**Описание**: Bottom navigation bar с 3 вкладками: «Документы», «Чат», «Настройки». Каждая вкладка — полноэкранный view. Выбор LLM — в «Настройках».

**Стиль**: Material Design 3, динамические цвета, скруглённые карточки.

**Преимущества**:
- Оптимально для мобильных устройств
- Простая навигация одним пальцем
- Быстрая загрузка (меньше DOM)

**Недостатки**:
- На десктопе выглядит пусто
- Переключение между документами и чатом требует лишних тапов

---

## Выбранная концепция: Концепция 1 (Split-Panel)

**Обоснование**: MVP ориентирован на продуктивную работу с документами. Split-Panel обеспечивает наилучший баланс между доступностью документов и фокусом на диалоге. На мобильных панели переключаются через нижний tab-bar (адаптация). Минималистичный стиль снижает когнитивную нагрузку и ускоряет разработку.

**Адаптация для мобильных**: при ширине < 768px левая панель скрывается, доступна через кнопку-гамбургер или нижний tab-bar.

## Задание для агента (frontend-design skill)

Сгенерировать React + TypeScript компоненты для Концепции 1:

1. `AppLayout` — двухпанельный layout с responsive breakpoints
2. `DocumentPanel` — список документов, кнопка загрузки, статус обработки
3. `ChatPanel` — история сообщений, поле ввода, индикатор загрузки
4. `LLMSelector` — dropdown выбора модели в header
5. `SourceCard` — карточка источника (имя файла, страница, релевантный фрагмент)

Стиль: Tailwind CSS, цвет акцента `blue-600`, шрифт Inter, тёмная тема опционально.

## Техническая реализация UI (утверждено)

- Framework: `Next.js` (App Router) + `TypeScript`
- Styling: `Tailwind CSS`
- Integration target:
  - runtime API: `FastAPI` backend endpoints
  - build output: публикация в `rag_app/backend/static/next` и `rag_app/backend/templates`

### Состояния компонентов

- `DocumentPanel`:
  - `loading`: загрузка/парсинг документа
  - `error`: ошибка формата/лимита/сети
  - `success`: документ готов, видны чанки/страницы
- `ChatPanel`:
  - `loading`: индикатор ответа ассистента
  - `error`: системное сообщение и действие повтора
  - `success`: ответ и карточки источников
- `LLMSelector`:
  - `loading`: загрузка списка моделей
  - `error`: провайдер недоступен
  - `success`: модель выбрана и применена

### Адаптивные правила

- `>= 1024px`: классический split-panel, документы слева, чат справа
- `768px - 1023px`: уплотненный split-panel, сокращенные метаданные
- `< 768px`: переключение между документами и чатом через tab-bar/drawer

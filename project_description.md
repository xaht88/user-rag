# RAG Веб-приложение — Описание продукта

## Проблема

LLM галлюцинирует при недостатке контекста или при работе с большими объёмами данных. Подход RAG (Retrieval-Augmented Generation) нивелирует этот эффект: по подготовленной векторной БД с помощью модели типа BERT выполняется поиск релевантных фрагментов по запросу пользователя, и через выбранную LLM предоставляется точный ответ.

## Ценность продукта

- Сокращает время на изучение объёмных документов
- Устраняет галлюцинации LLM за счёт заземления на реальный контекст
- Не требует специальных технических знаний от пользователя

## Целевая аудитория

- Пользователи ПК и мобильных устройств
- Специалисты, работающие с большим объёмом документации (юристы, аналитики, исследователи)
- Разработчики, изучающие кодовые базы и технические документы

## Ключевые функции (MVP)

1. **Выбор LLM** — пользователь выбирает языковую модель из доступных (OpenAI, Ollama)
2. **Загрузка документов** — загрузка PDF/DOCX/TXT/MD с автоматической обработкой в эмбеддинги
3. **Чат-интерфейс** — диалог с документами, отображение источников и истории

## Технический стек (рекомендуемый)

- **Backend**: Python (FastAPI), LangChain / LlamaIndex
- **Embedding**: sentence-transformers (BERT-based)
- **Vector Store**: ChromaDB / FAISS
- **Frontend**: React + TypeScript или Next.js
- **LLM провайдеры**: OpenAI API, Ollama (локально)

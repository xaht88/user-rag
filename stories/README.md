## Personas

### Аналитик (Analyst)
- Работает с большим объёмом PDF-отчётов
- Нужны быстрые ответы на конкретные вопросы
- Ценит точность и источники информации

### Разработчик (Developer)
- Изучает техническую документацию
- Хочет задавать вопросы по коду и архитектуре
- Ценит гибкость и контроль

### Мобильный пользователь (Mobile User)
- Работает с телефона
- Нужен простой и быстрый интерфейс
- Ценит удобство и адаптивность

## Story Categories

### Must Have (MVP)

Критические функции для запуска продукта:

- **US-01:** Загрузка документа
- **US-02:** Выбор языковой модели
- **US-03:** Диалог с документом
- **US-04:** Просмотр истории диалога

### Should Have

Важные функции для улучшения UX:

- **US-05:** Управление документами
- **US-06:** Выбор документов для поиска
- **US-07:** Ввод API-ключа

### Nice to Have

Желаемые функции для будущего:

- **US-08:** Мобильный интерфейс
- **US-09:** Отображение источников

### Frontend Delivery Stories

Специфичные для frontend реализации:

- **US-FE-10:** Состояния интерфейса компонента документов
- **US-FE-11:** Отправка запроса в чат с состояниями
- **US-FE-12:** Выбор LLM в рамках сессии

---

## Story Format

Каждая история содержит:

1. **ID** — Уникальный идентификатор
2. **Заголовок** — Краткое описание функции
3. **Как [персона]** — Кто использует функцию
4. **Я хочу [действие]** — Что нужно сделать
5. **Чтобы [цель]** — Зачем это нужно
6. **Критерии приёмки** — Конкретные условия выполнения

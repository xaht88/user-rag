# User Story Mapping: RAG Веб-приложение

## Персоны

- **Аналитик** — работает с большим объёмом PDF-отчётов, нужны быстрые ответы на конкретные вопросы
- **Разработчик** — изучает техническую документацию, хочет задавать вопросы по коду и архитектуре
- **Мобильный пользователь** — работает с телефона, нужен простой и быстрый интерфейс

---

## Must Have (MVP)

### US-01: Загрузка документа

Как **аналитик**, я хочу **загрузить PDF-документ**, чтобы **задавать вопросы по его содержимому**.

**Критерии приёмки**:
- Given: пользователь на главном экране
- When: выбирает файл PDF ≤ 50 МБ и нажимает «Загрузить»
- Then: файл обрабатывается, отображается прогресс-бар, по завершении — статус «Готов» и количество чанков

- Given: пользователь загружает файл > 50 МБ
- When: нажимает «Загрузить»
- Then: отображается ошибка «Файл превышает допустимый размер 50 МБ»

- Given: пользователь загружает файл неподдерживаемого формата (например, .xlsx)
- When: нажимает «Загрузить»
- Then: отображается ошибка «Формат не поддерживается. Допустимые форматы: PDF, DOCX, TXT, MD»

---

### US-02: Выбор языковой модели

Как **разработчик**, я хочу **выбрать LLM из списка**, чтобы **использовать нужную модель для ответов**.

**Критерии приёмки**:
- Given: пользователь открыл приложение
- When: нажимает на dropdown выбора LLM
- Then: отображается список доступных моделей (минимум GPT-4o-mini и одна локальная Ollama)

- Given: пользователь выбрал модель
- When: отправляет запрос
- Then: запрос обрабатывается выбранной моделью, название модели отображается в интерфейсе

- Given: выбранная LLM недоступна (нет API-ключа или сервер Ollama не запущен)
- When: пользователь пытается отправить запрос
- Then: отображается ошибка с предложением выбрать другую модель

---

### US-03: Диалог с документом

Как **аналитик**, я хочу **задать вопрос по загруженному документу**, чтобы **получить точный ответ со ссылкой на источник**.

**Критерии приёмки**:
- Given: документ загружен и обработан, LLM выбрана
- When: пользователь вводит вопрос и нажимает «Отправить»
- Then: в течение 30 секунд отображается ответ LLM с указанием источников (имя файла, страница)

- Given: пользователь отправил запрос
- When: система обрабатывает запрос
- Then: отображается индикатор загрузки

- Given: документы не загружены
- When: пользователь пытается отправить запрос
- Then: отображается подсказка «Загрузите документ для начала работы»

---

### US-04: Просмотр истории диалога

Как **аналитик**, я хочу **видеть историю вопросов и ответов**, чтобы **не повторять уже заданные вопросы**.

**Критерии приёмки**:
- Given: пользователь задал несколько вопросов
- When: прокручивает чат вверх
- Then: отображается полная история диалога текущей сессии

---

## Should Have

### US-05: Управление документами

Как **аналитик**, я хочу **удалять ненужные документы**, чтобы **контролировать контекст поиска**.

**Критерии приёмки**:
- Given: список документов содержит загруженные файлы
- When: пользователь нажимает «Удалить» рядом с документом
- Then: документ и его чанки удаляются из Vector Store в течение 5 секунд, список обновляется

- Given: пользователь удалил все документы
- When: удаление завершено
- Then: история диалога очищается, отображается уведомление

---

### US-06: Выбор документов для поиска

Как **разработчик**, я хочу **выбирать, по каким документам искать**, чтобы **получать более точные ответы**.

**Критерии приёмки**:
- Given: загружено несколько документов
- When: пользователь снимает галочку с одного документа
- Then: следующие запросы не используют этот документ для поиска

---

### US-07: Ввод API-ключа

Как **разработчик**, я хочу **ввести свой API-ключ OpenAI**, чтобы **использовать собственный аккаунт**.

**Критерии приёмки**:
- Given: пользователь выбрал модель OpenAI
- When: вводит API-ключ в поле настроек
- Then: ключ используется для запросов текущей сессии и не сохраняется на сервере

---

## Nice to Have

### US-08: Мобильный интерфейс

Как **мобильный пользователь**, я хочу **удобно работать с приложением на телефоне**, чтобы **использовать его в любом месте**.

**Критерии приёмки**:
- Given: пользователь открывает приложение на телефоне (ширина < 768px)
- When: страница загружается
- Then: интерфейс адаптирован под мобильный экран, все элементы доступны

---

### US-09: Отображение источников

Как **аналитик**, я хочу **видеть точный фрагмент текста из документа**, чтобы **проверить достоверность ответа**.

**Критерии приёмки**:
- Given: LLM дала ответ на основе документа
- When: пользователь нажимает на источник
- Then: отображается релевантный фрагмент текста с указанием страницы

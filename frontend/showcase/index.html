User Story Mapping

9 историй · 3 приоритета · покрытие всех ключевых функций MVP

Must Have
US-01 · Загрузка документа
Как аналитик, я хочу загрузить PDF, чтобы задавать вопросы по содержимому.
Given файл ≤50МБ When нажимает «Загрузить» Then прогресс-бар → статус «Готов»
US-02 · Выбор LLM
Как разработчик, я хочу выбрать модель, чтобы получать ответы нужного качества.
Given открыт dropdown When выбирает GPT-4o-mini Then модель применяется к запросам
US-03 · Диалог с документом
Как аналитик, я хочу задать вопрос, чтобы получить ответ со ссылкой на источник.
Given документ загружен When отправляет вопрос Then ответ ≤30с + источники
US-04 · История диалога
Как аналитик, я хочу видеть историю, чтобы не повторять вопросы.
Given задано несколько вопросов When прокручивает вверх Then полная история сессии
Should Have
US-05 · Управление документами
Как аналитик, я хочу удалять документы, чтобы контролировать контекст поиска.
Given список документов When нажимает «Удалить» Then удаление ≤5с, список обновляется
US-06 · Выбор документов
Как разработчик, я хочу выбирать документы для поиска, чтобы получать точные ответы.
Given несколько документов When снимает галочку Then документ исключён из поиска
US-07 · API-ключ
Как разработчик, я хочу ввести API-ключ OpenAI, чтобы использовать свой аккаунт.
Given выбран OpenAI When вводит ключ Then ключ в памяти сессии, не сохраняется
Nice to Have
US-08 · Мобильный интерфейс
Как мобильный пользователь, я хочу работать с телефона, чтобы использовать приложение везде.
Given ширина <768px When загружает страницу Then адаптивный layout
US-09 · Источники
Как аналитик, я хочу видеть фрагмент текста из документа, чтобы проверить ответ.
Given получен ответ When нажимает на источник Then фрагмент + номер страницы

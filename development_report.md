## 3. Описание проблем и их решений

### Проблема 1: Конфликт технологий (Next.js vs Vite)
**Описание:**
- package.json использовал Next.js скрипты (`next dev`, `next build`)
- Документация указывала на Vite + React
- Порт по умолчанию Next.js (3000) не совпадал с Vite (5173)

**Причина:**
- Изначально проект планировался как Next.js, но позже принято решение использовать Vite
- Не все файлы были обновлены в процессе миграции

**Решение:**
1. Обновлен package.json с Vite скриптами:
   ```json
   {
     "scripts": {
       "dev": "vite",
       "build": "tsc && vite build",
       "preview": "vite preview"
     }
   }
   ```
2. Создан `vite.config.ts` с правильной конфигурацией
3. Создан `index.html` как точка входа
4. Обновлены все переменные окружения (NEXT_PUBLIC_* → VITE_*)
5. Обновлена документация (AGENTS.md, docs/)

**Результат:** Frontend успешно работает на Vite + React

---

### Проблема 2: Дублирование файлов в backend
**Описание:**
- В директории `rag_app/backend/` находились 3 файла с похожим функционалом:
  - `main.py` — текущая реализация API
  - `rag_app.py` — устаревшая реализация
  - `_rag_app.py` — временный файл

**Причина:**
- В процессе разработки создавались новые версии файлов
- Старые файлы не были удалены
- Не было единого источника истины

**Решение:**
1. Проанализирован код всех файлов
2. Определено, что `main.py` содержит актуальную реализацию
3. Удалены `rag_app.py` и `_rag_app.py`
4. Обновлен AGENTS.md с актуальной структурой
5. Создан ADR-003 для документирования изменений

**Результат:** Чистая структура backend без дубликатов

---

### Проблема 3: Ошибки компиляции в компонентах
**Описание:**
- Ошибка `Duplicate export error` в файле `chat-panel.tsx`
- Дублирующиеся импорты и экспорты в одном файле

**Причина:**
- При генерации кода AI-агентом были созданы дублирующиеся секции
- Не было проверки на дубликаты перед финализацией

**Решение:**
1. Прочитан файл `chat-panel.tsx` полностью
2. Найдены дублирующиеся экспорты функции `ChatPanel`
3. Удалены дублирующиеся импорты и секции кода
4. Проверены другие компоненты на аналогичные проблемы

**Результат:** Ошибка компиляции устранена, проект собирается успешно

### Проблема 4: Несоответствие API endpoints
**Описание:**
- Frontend вызывал `/llm/config`, но backend использовал `/api/sessions/{session_id}/llm/config`
- Несоответствие между клиентом и сервером

**Причина:**
- API endpoints изменились в процессе разработки backend
- Frontend не был обновлен своевременно

**Решение:**
1. Проанализированы endpoints в `main.py`
2. Обновлен `client.ts` с правильными путями:
   ```typescript
   async queryLLM(sessionId: string, query: string, documentIds?: string[]) {
     const res = await fetch(`${BASE_URL}/api/sessions/${sessionId}/query`, {
       method: 'POST',
       body: JSON.stringify({ session_id: sessionId, query, document_ids: documentIds })
     });
   }
   ```
3. Обновлены импорты в `page.tsx`

**Результат:** Frontend и backend синхронизированы

---

### Проблема 5: Отсутствие Python окружения для backend
**Описание:**
- При попытке запустить backend команда `python -m uvicorn` не находила Python
- Virtual environment не был активирован корректно

**Причина:**
- Python не установлен в системе или не добавлен в PATH
- Virtual environment требует ручной активации

**Решение:**
1. Проверено наличие Python в venv: `.\venv\Scripts\python.exe`
2. Подготовлена инструкция по запуску с активацией venv
3. Для тестирования frontend запущен без backend (mock-данные)

**Результат:** Frontend работает с mock-данными, backend требует настройки Python

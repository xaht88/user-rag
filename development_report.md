# Development Report

## 1. Цель

Реализовать frontend-часть ДЗ на `Next.js + Tailwind` с документированной работой AI-агента, тестированием и адаптивной структурой.

## 2. Что сделано

1. Синхронизированы спецификации:
   - `technical_specification.md`
   - `stories/user_stories.md`
   - `frondent/ui_description.md`
2. Подготовлена спецификация frontend:
   - `frontend_specification.md`
3. Инициализирован frontend-каркас:
   - `rag_app/frontend_next/*`
4. Реализованы ключевые UI-компоненты:
   - `AppLayout`, `DocumentPanel`, `ChatPanel`, `LLMSelector`, `SourceCard`
5. Подготовлены автотесты (3+).

## 3. Примеры промптов и результаты

- Промпт: "Сначала предложи план, затем создавай/дополняй спецификации после одобрения."
  - Результат: согласованный план работ и переход к реализации.
- Промпт: "Implement the plan as specified... mark todos in_progress/completed."
  - Результат: последовательная реализация задач по todo-списку с фиксацией статусов.

## 4. Проблемы и решения

- Проблема: в репозитории отсутствовали `README.md`, `development_report.md`, `package.json`.
  - Решение: добавлены новые артефакты и frontend-каркас.
- Проблема: интеграция Next.js с FastAPI требует отдельного flow для dev/prod.
  - Решение: добавлен скрипт `scripts/export-to-backend.ps1` и описание pipeline.
- Проблема: часть backend UI реализована как legacy/stub.
  - Решение: source of truth перенесен в `rag_app/frontend_next`, legacy не удалялся.

## 5. Техники AI-отладки

- Анализ ошибок компиляции и runtime по сообщениям тестов/консоли.
- Итеративная правка компонент по обратной связи тестов.
- Для визуальных багов рекомендуется:
  - прикладывать скриншоты состояния UI;
  - передавать их AI для определения расхождений с макетом;
  - генерировать targeted-fix по конкретному компоненту.

## 6. Эффективные практики

- Сначала стабилизировать спецификации, затем код.
- Дробить UI на изолированные компоненты и покрывать их тестами.
- Разделять mock/live режимы через feature-слой.

## 7. Выводы и рекомендации

- Подход "спецификация -> каркас -> компоненты -> тесты -> отчеты" снижает риск доработок.
- Для следующей итерации полезно добавить e2e smoke (`Playwright`) и CI.
